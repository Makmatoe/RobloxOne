SessionDock 2.6.2

Security-hardened releases and integrations

- Updated the self-contained Windows runtime to .NET 10.0.10 and the build SDK to 10.0.302, with automatic checks for future security patches.
- Strengthened Roblox executable signature validation with online certificate revocation checks and fresh validation before sensitive actions.
- Made one-click HandleScope installation work with its current immutable GitHub release while verifying GitHub asset digests, the matching checksum, safe archive layout, every bundled file, and the installed API hash.
- Kept the update descriptor, exact package hashes, SBOM, checksums, GitHub attestations, draft re-download, and separate approval gates while documenting that the Windows executables are currently unsigned.
- Removed the isolated smoke-test path from production builds and strengthened dependency, workflow, and repository policy checks.

HandleScope and SessionDock are currently unsigned, so Windows may show Unknown publisher. Their checksum and release verification do not provide certificate-backed publisher identity.
