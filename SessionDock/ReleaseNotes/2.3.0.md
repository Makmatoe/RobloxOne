# SessionDock 2.3.0

## Safer HandleScope setup

- A new **Integrations** panel guides users through the optional HandleScope
  connection without requiring PowerShell.
- SessionDock can explicitly start only the separately installed API at its
  expected per-user path after local safety checks, enable or disable the fixed
  Roblox policy, and test its loopback health endpoint.
- Opening or refreshing the panel performs local file checks only. Starting
  the API and testing its connection always require separate user actions.
- Existing custom or invalid integration settings are preserved. SessionDock
  offers repair only after showing a warning, and repair writes only the
  minimal enabled or disabled opt-in.
- Status results never expose HandleScope tokens, ports, process IDs, or local
  paths. The connection test never enumerates or closes handles.

This release also removes the last visible Roblox One branding from the main
window. HandleScope remains a separate, unsigned installation and is never
bundled, downloaded, elevated, or silently started by SessionDock. Install it
only from the official HandleScope release page and verify its published
checksum.
