SessionDock 2.3.2

Recovery and release safety

- Recovery now protects browser profiles when account metadata or pending cleanup state cannot be recovered safely.
- A hard shutdown deadline prevents stalled background work from keeping SessionDock alive indefinitely.
- Advanced generic launch hooks now require a trusted HTTPS numeric-loopback endpoint and a bearer token. HandleScope integration is unchanged.
- Continuous integration and protected release builds now launch the isolated published application before accepting its output.
- A verified draft must pass a separate approval before publication, followed by another inventory, checksum, and GitHub attestation check.
