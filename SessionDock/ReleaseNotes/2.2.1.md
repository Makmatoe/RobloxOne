# SessionDock 2.2.1

## Per-account batch destinations

- Every selected account now launches its own saved public experience,
  private-server link or code, or tracked Server Job ID.
- A selected account with no saved destination uses the first selected
  account's destination for that batch only.
- The first selected account must have a valid destination, and SessionDock
  validates every destination before it closes any running Roblox client.
- The batch window previews each account's destination before launch.
- Batch launching no longer overwrites saved account destinations.

This patch adds no dependencies and does not change the local account or
HandleScope security boundaries.
